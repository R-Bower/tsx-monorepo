# Props API

## Space

The space utility converts shorthand margin and padding props to margin and padding CSS declarations.

- Numbers from 0 to the length of `theme.space` are converted to pixel values.
- Negative values can be used for negative margins.
- Numbers greater than the length of the `theme.space` array are converted to raw pixel values.
- String values are passed as raw CSS values.
- Array values are converted into [responsive values](/core-concepts#responsive-props).

Margin and padding props follow a shorthand syntax for specifying direction.

## Color

The color utility parses a component's `color` and `bg` props and converts them into CSS declarations.  Color palettes are configured with the ThemeProvider to use keys as prop values, with support for dot notation.

```jsx
// picks up a nested color value using dot notation
// `theme.colors.text.primary`
<Text color={"text.primary"} />

// Raw CSS color value
<Box bg={"#FFFFFF"} />

// verbose prop
<Box backgroundColor={"#FFFFFF"} />
```

### Sizing
The sizing group includes style props for width, height, minWidth, minHeight, maxWidth, maxHeight, and size.

Each prop is transformed based on the following:

- Numbers from 0-1 are converted to percentage widths.
- Numbers greater than 1 are converted to pixel values.
- String values are passed as raw CSS values.
- Arrays are converted to [responsive values](/core-concepts#responsive-props).
- If theme.sizes is defined, the width prop will attempt to pick up values from the theme.

```jsx
// examples
// width `50%`
<Box width={1/2} />

// width `256px`
<Box width={256} />

// width `'2em'`
<Box width={"2em"} />

// width `100%` on all viewports and `50%` from the smallest breakpoint and up
<Box width={[1, 1/2]} />

// width from `theme.sizes`
<Box width={"medium"} />

// size (width & height)
<Box size={32} />
<Box size={[32, 48]} />
```
